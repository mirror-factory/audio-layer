# Layer One — Brand Guide

## Personality

**Precise. Effortless. Alive.**

- **Precise:** Every element has a reason. Nothing decorative, nothing wasted.
- **Effortless:** Feels like it anticipates what you need. Low friction, high clarity.
- **Alive:** Responsive, interactive, things move and react. Not static, not stale.

## Visual Identity

### Color

- **Primary accent:** Mint (#14b8a6). Used sparingly — active states, key actions, recording indicators. Mint means "actionable."
- **Background:** Dark (#0a0a0a). Not gamer-dark — refined. Think Vercel meets Linear, with warmth.
- **Surfaces:** Subtle depth through tinted neutrals (#171717 cards on #0a0a0a), not drop shadows or borders.
- **Semantic:** Green = success, Red = error, Yellow = warning, Blue = info/user messages.

### Typography

- Monospace throughout — this is the app's identity.
- Font stack: `ui-monospace, SFMono-Regular, "SF Mono", Menlo, Consolas, monospace`
- Headlines: 600 weight. Body: 400. Labels: 500 uppercase.

### Motion

- Purposeful. Page transitions, recording pulse, status changes.
- Not bouncy, not flashy — smooth and confident.
- Respect `prefers-reduced-motion: reduce`.

## Anti-References

- Generic SaaS dashboards
- Material UI defaults
- Gradient text, glassmorphism, neon-on-dark
- Anything that looks like "a React template"

## References

- Linear (precision)
- Raycast (speed)
- Granola (warmth)
- Vercel (technical confidence)

## Design Principles

1. **Content density over decoration** — show more useful information, less chrome
2. **Interaction over ornamentation** — things should feel alive when you touch them
3. **One accent color, used with discipline** — mint means "active, important, actionable"
4. **Progressive disclosure** — simple at first glance, powerful on exploration
5. **Cross-platform coherence** — same design language on web, macOS, and iOS
